# Output Template — 7-Section Standardized Response

## 格式（每個 case 都用以下 7 段）

---

### 📋 1. 病例 Summary
（一段重述：年齡、GA、病史、用藥、關鍵 lab、症狀、iodine 攝取；50–80 字）

> ⚠️ Missing data：[列出影響決策的缺失欄位，若無則略]

---

### 🎯 2. Case Archetype
- 對應 Decision Tree：A / B / C / D / E / F / G（描述）
- 若同時 fit 多型，標明所有相關 archetype

---

### 🧠 3. 對應背景知識
列出相關 knowledge file 條目，引用具體數字 / RCT / NNT。例：
- TSH 5.2 mIU/L 在 trimester upper limit 之上
- 丹麥 cohort 顯示 ~50% 自然 normalize（Eur Thyroid J 2022;11:e210055）
- 2024 ASRM：不建議 routine LT4 for SCH

---

### 🔍 4. 整合分析（2–3 段）
**第一段**：本案核心鑑別 / 病理生理機轉
**第二段**：證據強度 + 治療效益風險
**第三段**：本案個人化決策（為何選 X 不選 Y）

---

### 💊 5. 建議下一步
| 項目 | 建議 |
|---|---|
| **Lab** | 列出要驗的檢驗 + 時程 |
| **藥物** | 起始 / 增減 / 停藥 + 劑量 |
| **轉診** | Endo / MFM / REI / Genetic counseling 等 |
| **下次回診** | 時程 + 看哪些指標 |
| **Fetal monitoring** | US 時點 + 項目（若 Graves / TRAb+） |
| **Neonatal plan** | Cord blood / heel prick adjustment（若需要） |
| **Iodine** | 補充劑量 + 持續期間 |

---

### 🗣️ 6. 病人衛教 / SDM 話術（中文，可直接讀給病人）
> 三段式：
> 1. **正常化**：（把臨床數字框架成常見、可理解）
> 2. **數字框架**：（給具體比例 / 機率，避免絕對化）
> 3. **計畫式語言**：（把下一步講成 plan，避免 wait）

---

### ⚠️ 7. 安全提醒 / Escalate Triggers
- 列出何時應立即 escalate to attending
- 列出本決策 NOT 適用的灰區
- 列出本案 missing data 中、若值落在某範圍就會改變建議的 cutoff

---

### ❓ Open Questions（2–3 條）
- 列出建議臨床醫師額外考慮的問題

---

## ⚠️ Disclaimer（每次都附）
> 本輸出為臨床決策**支援工具**（CDS），不取代主治醫師的判斷。所有建議須與病人臨床全貌、最新指引、機構政策對照後再執行。對 hard-stop 觸發條件，請直接請主治。
